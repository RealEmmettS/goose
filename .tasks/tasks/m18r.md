TT;DR: Implement and validate native Wayland reduced mode: visible layer-shell overlay, local IPC control, and explicit unsupported mischief behavior.

## Status

Done - Linux x64 and ARM CI smoke passed and the run URL/evidence is recorded.

## Why

Native Wayland deliberately blocks global cursor warp, synthetic input, and portable foreign-window move/enumeration. M18 should still render and remain controllable while reporting those tricks as unsupported.

## Acceptance

- `--wayland` starts a visible reduced-mode overlay on at least one supported Wayland compositor path.
- `honk300 status`, reload, stop, and direct honk/mud/wander pokes work through Unix IPC.
- Cursor nab, foreign-window ride, collect-window move/type, and keystroke synthesis report unsupported instead of crashing or hanging.
- Terminal protection assumptions are documented for any compositor-specific metadata used later.
- `script/smoke_m17_m18_linux.sh` passes for default and forced-Wayland degraded control behavior.
- Linux-host CI smoke records compositor/session details and visible-pixel proof.
